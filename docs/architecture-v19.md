# asset-app v20 运维体验增强版

v20 针对三个方向优化：版本号显示、数据源网址变更记录、同步结果对比。

## 1. 版本号显示

系统会在侧边栏和系统维护页面显示当前版本与构建时间：

- `APP_VERSION`，默认 `v20`
- `APP_BUILD`，默认 `2026-06-22`

可以在 `.env` 中覆盖：

```env
APP_VERSION=v20
APP_BUILD=2026-06-22
```

## 2. 数据源网址变更记录

当你修改数据源网址时，系统会自动记录：

- 数据源 ID
- 数据源名称
- 旧网址
- 新网址
- 操作时间
- 操作人
- 变更原因

页面路径：数据源 → 选择数据源 → 下方“网址变更记录”。

## 3. 误改网址后的处理逻辑

如果你把某个数据源网址改错了，例如：

```text
万盛｜管城
原网址：https://boxin888.cc/
误改成：https://wrong.example/
```

处理方式：

1. 进入“数据源”页面。
2. 选择对应的数据源。
3. 在“网址变更记录”里找到正确的旧网址。
4. 点击“回滚到旧网址”。

回滚时系统会：

- 把当前数据源网址恢复到旧网址。
- 把误改的网址也保留为别名，避免历史资产失去分类。
- 重新执行历史资产绑定修复。
- 不会删除你的资产数据。

因此后续即使网址多次变化，只要数据源 ID 没被删除，历史资产仍然会跟着“标头｜标签”走，而不是跟着网址走。

## 4. 同步结果对比

同步完成后，系统会记录并显示：

- 解析数量
- 新增数量
- 更新数量
- 纯净转风控数量
- 风控转纯净数量
- 状态变化数量
- 未变化数量
- 本次未出现而转为纯净的数量

这样可以判断一次同步是不是异常，例如突然大量“纯净转风控”或者大量“失效转纯净”。
